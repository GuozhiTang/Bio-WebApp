FirePlex Web Application

It is coded by using MEAN Stack(MongoDB, Express, Angular-CLI, Nodejs).

For the time being, these main components are created: Login, Dashboard, System, Generate Requests, Components and Data

Login

Registered users can login to the application by:

1. Login selection at navbar

2. Login button in home page

Only operator (user) name is required to login.

Once login successfully, the login information will be kept in local storage for a week unless logging out of system.

Dashboard

It shows basic information for an operator:

1. Name and EmployeeId

2. Associated department(s) and related Ids

The departments information for some operators could be empty which means they only have access to modify basic documents in the system.

System

It contains some basic functions in system level:

1. Operator+

  • Register new operators
  • Display and synchronize operator information from data server
  • Only operator (user) name is required to register.
  • Operators can only be registered by existed operator.

2. Logout

  • Logout of the application to the login page
  • Clear the login and operator information in the local storage
  • Operators can logout by access the botton in homepage or the selection in "System" at navbar.
  • If not, the information will be kept login for a week.

Generate Requests

To generate different types of requests:

1. Code Dillution Request

2. Code Mix Request

3. Instrument Request

4. Transfer Request

5. Matrix Tube Carrier Request

  • Associated and specific departemnt is required. If not, the operator has no authentication.

    • Components

    Some individual components with different functioanlities:

    1. Setup Process: To setup and display the work-orders.

    • Show the list of requests and meanwhile the details of each work-order
    • Setup necessary information for each work-order
    • Display and update status for each work-order
    • Associated and specific departemnt is required. If not, the operator has no authentication.

    2. Transfer:

    • An old sample process for creating the transfer request
    • A sample for plate customization which can be edited and displayed in well-level of a plate

    Data

    • It shows all the details for some tables (data) in database locally and remotely
    • It provide the function to synchronize these data from the data server
    • It provides the function to search for specific data with resonable conditions
    • It provides the function to create new data at the same time for local database and remote data server
    • Create duplicated data is denied for the function.
    • All the new functions are not completely applied for "Probemaps" data component.

    These data are included:

    1. Roles

    2. Instruments

    3. LabwareSpecs

    4. Probemaps
